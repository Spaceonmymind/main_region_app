# Запуск приветственного чат-бота

Проект содержит простой сценарий чат-бота MAX:

- приветствует пользователя при начале диалога;
- отвечает на команду `/start`;
- объясняет назначение мини-приложения;
- показывает кнопку «Открыть сервисы».

## Подготовка

1. Создайте бота в MAX для партнёров и дождитесь прохождения модерации.
2. Получите токен в разделе:
   `Чат-боты → Перейти → Расширенные настройки → Настроить`.
3. Скопируйте `.env.example` в `.env`.
4. Заполните `BOT_TOKEN`.
5. Оставьте `PORT=3001` или укажите другой свободный порт.
6. Чтобы включить админскую статистику, напишите боту команду `/my_id`,
   скопируйте полученный MAX user_id и добавьте его в `.env`:
   `ADMIN_USER_IDS=123456`.
7. В настройках бота привяжите HTTPS-адрес собранного мини-приложения:
   `https://service-bot.vi.gov45.ru`.
8. В `.env` сохраните этот адрес в `PUBLIC_MINI_APP_URL`:
   `PUBLIC_MINI_APP_URL=https://service-bot.vi.gov45.ru`.
9. В `.env` заполните `MINI_APP_DEEPLINK` диплинком MAX:
   `https://max.ru/<botName>?startapp`.

`PUBLIC_MINI_APP_URL` — это место, где лежит мини-приложение.
Именно его нужно указать в MAX для партнёров:
`Чат-боты → Перейти → Расширенные настройки → Настроить → URL мини-приложения`.

`MINI_APP_DEEPLINK` — это ссылка для кнопки «Открыть сервисы» в сообщении бота.
Она должна вести на `https://max.ru/<botName>?startapp`, тогда MAX откроет
привязанное мини-приложение внутри себя, а не просто перебросит пользователя
на внешний сайт.

Без `MINI_APP_DEEPLINK` приветственные сообщения работают, но кнопка запуска
не отображается. Для совместимости старое имя `MINI_APP_URL` пока тоже
поддерживается, но лучше использовать `MINI_APP_DEEPLINK`.

## Админская статистика

Бот поддерживает команды:

- `/my_id` — показать ваш MAX user_id;
- `/admin_stats` или `/stats` — показать статистику, доступно только id из `ADMIN_USER_IDS`.

Статистика хранится на сервере в файле `data/analytics.json` и переживает
перезапуск systemd-сервиса.

Собираются:

- запуски бота и команды `/start`;
- сообщения пользователей боту;
- открытия мини-приложения;
- просмотры страниц;
- клики по категориям;
- переходы к карточкам услуг;
- нажатия «Открыть» у услуг;
- поисковые запросы и поиски без результата.

Файл `data/analytics.json` не коммитится в git.

## Запуск

```bash
yarn bot:start
```

Вместе с ботом запускается HTTP-сервер на `0.0.0.0:$PORT`:

- `GET /health` — проверка состояния сервиса;
- `POST /webhook` — приём и логирование JSON-событий MAX.
- `POST /analytics/event` — приём событий мини-приложения для админской статистики;
- остальные `GET`-запросы — файлы собранного мини-приложения из `public/`.

Пример проверки:

```bash
curl http://127.0.0.1:3001/health
curl -X POST http://127.0.0.1:3001/webhook \
  -H 'Content-Type: application/json' \
  -d '{"update_type":"test"}'
```

Текущая логика ответов бота продолжает использовать получение обновлений
официальной библиотекой. Endpoint `/webhook` пока только принимает и логирует события.
